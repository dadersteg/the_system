# Credential Rotation & Revocation Guide (Task T2)

This document outlines the step-by-step procedure required to rotate and revoke all exposed credentials for **The System**. Since some of these operations require web console login or password changes, they must be executed manually by the user.

---

## 1. Facebook & Messenger Session Invalidation
*   **Problem:** Live session cookies were committed in `Fca_Database/database.sqlite` and `src/ingestion/Fca_Database/database.sqlite`.
*   **Actions:**
    1.  Go to **Facebook Settings & Privacy** > **Security and Login**.
    2.  Change your Facebook password. *Choose the option to "Log out of other devices" when prompted.* This invalidates all existing session cookies globally.
    3.  Clear local database cache files by running:
        ```bash
        python3 scripts/utils/clear_compromised_sessions.py
        ```

---

## 2. Google OAuth Credentials & Tokens
*   **Problem:** Google OAuth client ID and client secret are exposed in `auth/credentials.json`.
*   **Actions:**
    1.  Go to the **Google Cloud Console** (https://console.cloud.google.com/).
    2.  Select project `the-system-1234` (or the project associated with the credentials).
    3.  Navigate to **APIs & Services** > **Credentials**.
    4.  Under **OAuth 2.0 Client IDs**, edit the client ID used by the system.
    5.  Click **Reset Secret** to generate a new OAuth client secret.
    6.  Download the updated client credentials as a JSON file and replace:
        *   `auth/credentials.json`
        *   `auth/creds.json`
        *   `auth/creds_work.json`
    7.  Revoke all existing authorization grants:
        *   Go to **Google Account Settings** > **Security** > **Your connections to third-party apps & services**.
        *   Remove access for **The System**.
    8.  Delete all existing token JSON files to force re-authentication:
        ```bash
        python3 scripts/utils/clear_compromised_sessions.py
        ```
    9.  Re-run the scripts manually once to trigger the interactive OAuth flow on the server and generate fresh tokens.

## 3. Telegram API Credentials & Bot Tokens
*   **Problem:** Telegram bot tokens, API ID, and API Hash are exposed in `.env`.
*   **Actions:**
    1.  **Bot Tokens:**
        *   Open Telegram and start a chat with `@BotFather`.
        *   Send `/revoke` and select the primary bot token (`8624910336:AAFG0slF5ZiD2yCkHYEGykNsfEvKSj2QSak`).
        *   Send `/revoke` again and select the SMS relay bot token (`8971429996:AAFmJi46ZQ8fzTnDTxqqORnxCwiGSgX_G-w`).
        *   Copy the new tokens generated by `@BotFather` and update `.env`:
            ```env
            TELEGRAM_BOT_TOKEN="<new_token>"
            SMS_RELAY_BOT_TOKEN="<new_token>"
            ```
    2.  **API ID & Hash:**
        *   Log in to https://my.telegram.org/.
        *   Navigate to **API development tools**.
        *   Create a new application to obtain a new API ID and API Hash.
        *   Update `.env`:
            ```env
            TELEGRAM_API_ID='<new_api_id>'
            TELEGRAM_API_HASH='<new_api_hash>'
            ```
    3.  **Terminate Telethon Session:**
        *   Delete local session files by running `python3 scripts/utils/clear_compromised_sessions.py`.
        *   Upon next start of the Telegram bridge, a new OTP login flow will be prompted to re-authenticate securely.

## 5. Gemini API Keys
*   **Problem:** Google Gemini API keys are exposed in `.env` and `test_prompt3.py`.
*   **Actions:**
    1.  Go to **Google AI Studio** (https://aistudio.google.com/).
    2.  Select **Get API key** from the menu.
    3.  Delete/revoke all compromised keys (including `SYSTEM_GEMINI_API_KEY` and `WORK_GEMINI_API_KEY`).
    4.  Generate new API keys.
    5.  Update `.env`:
        ```env
        SYSTEM_GEMINI_API_KEY='<new_system_key>'
        WORK_GEMINI_API_KEY='<new_work_key>'
        ```

## 6. Jules API Key
*   **Problem:** `JULES_API_KEY` is exposed in `.env`.
*   **Actions:**
    1.  Rotate the Jules key in the developer console.
    2.  Update `.env`:
        ```env
        JULES_API_KEY='<new_jules_key>'
        ```
